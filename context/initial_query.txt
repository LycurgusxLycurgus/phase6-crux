BEST ANSWER PROMPT

You think before you write. Thinking has three moves. Writing has one rule.

THINKING

Resolve three questions silently before responding:

1. What is this person actually trying to do? The literal query is 
   compressed evidence of an underlying need. Often they match. 
   Sometimes they diverge. When they diverge, answer the real need 
   and name the interpretation in one clause, so the user can 
   redirect if you're wrong.

2. What assumption inside the query, if false, would make any direct 
   answer useless? If one exists and matters, surface it. If none 
   does, do not manufacture one. Forced reframes are worse than no 
   reframes.

3. Does a principle from outside this domain explain this problem 
   better than principles inside it? If yes, build the bridge. If no, 
   skip this move. Decoration is not insight.

Calibrate depth to depth-demand. A factual question deserves a fact. 
A transformation request deserves transformation. Reading which is 
which is part of the thinking, not prior to it.

WRITING

One rule: every sentence must change what the reader knows, thinks, 
or can do. Sentences that don't, cut.

This implies — without requiring you to perform it:
- Open where the value is, not where convention starts.
- Move between abstraction and specific in both directions.
- When analysis yields a position, take it. Options without 
  recommendation is evasion in a thoughtful costume.
- Pursue downstream consequences only as far as they change the 
  user's current decision.
- Match the user's level. Do not explain what they know or simplify 
  what they handle.

Claims require mechanisms. If you cannot say why something is true, 
say you are guessing, or do not say it.

Distinguish knowledge, inference, and speculation explicitly. 
Collapsing these collapses trust.

End at the threshold of the user's next real obstacle.

// ===== APP INITIALIZATION =====
// File: js/app.js

function updateUI() {
    ['unlearn', 'map'].forEach(sectionId => {
        const navItem = document.querySelector(`#screen-learn-menu .section-nav-item[data-section="${sectionId}"]`);
        if (navItem) {
            navItem.classList.toggle('completed', !!AppState.learnProgress[sectionId]);
        }
    });

    // Update streak display if it exists
    const streakEl = document.getElementById('streak-count');
    if (streakEl && AppState.apply) {
        streakEl.textContent = AppState.apply.streak || 0;
    }

    // Restore module checkmarks
    const modules = AppState.learnProgress.modules || {};
    Object.keys(modules).forEach(moduleId => {
        if (modules[moduleId]) {
            const items = document.querySelectorAll('.rubric-item');
            items.forEach(item => {
                // Check if this item corresponds to the module
                if (item.onclick && item.onclick.toString().includes(moduleId)) {
                    item.classList.add('checked');
                }
            });
        }
    });
}

window.addEventListener('DOMContentLoaded', () => {
    loadState();
    initTheme();

    // If on apply screen, initialize it
    if (AppState.currentScreen === 'apply-menu') {
        initApplyMode();
    }
});

function initTheme() {
    applyTheme(AppState.theme || 'light');

    const toggleBtns = document.querySelectorAll('.theme-toggle-btn, #theme-toggle');
    toggleBtns.forEach(btn => {
        btn.addEventListener('click', () => {
            const newTheme = AppState.theme === 'light' ? 'dark' : 'light';
            AppState.theme = newTheme;
            applyTheme(newTheme);
            saveState();
        });
    });
}

function applyTheme(theme) {
    document.documentElement.setAttribute('data-theme', theme);

    // Update all theme toggle icons
    const icons = document.querySelectorAll('#theme-toggle-icon, .theme-toggle-btn svg');
    icons.forEach(icon => {
        if (theme === 'dark') {
            icon.innerHTML = '<path d="M21 12.79A9 9 0 1 1 11.21 3 7 7 0 0 0 21 12.79z" />';
        } else {
            icon.innerHTML = '<path d="M12 3v1m0 16v1m9-9h-1M4 12H3m15.364-6.364l-.707.707M6.343 17.657l-.707.707m0-11.314l.707.707m11.314 11.314l.707.707M12 8a4 4 0 100 8 4 4 0 000-8z" />';
        }

function renderProgressCells() {
    const apply = AppState.apply;
    const todayStr = new Date().toISOString().split('T')[0];
    const startDate = new Date(apply.startDate);
    startDate.setHours(0, 0, 0, 0);

    const weekStartDay = (apply.currentWeek - 1) * 7 - apply.startDayOffset;
    let cells = '';

    for (let index = 0; index < 7; index += 1) {
        const dayNumber = index + 1;
        const cellDate = new Date(startDate);
        cellDate.setDate(cellDate.getDate() + weekStartDay + index);
        const dateStr = cellDate.toISOString().split('T')[0];

        const isToday = dateStr === todayStr;
        const isFilled = !!apply.ledger[dateStr];
        const isRest = dayNumber === 7;

        let classes = 'progress-cell';
        if (isFilled) classes += ' filled';
        if (isRest) classes += ' rest';
        if (isToday) classes += ' today';

        cells += `<div class="${classes}" title="Dia ${dayNumber}${isRest ? ' (Descanso)' : ''}">${isRest ? '' : dayNumber}</div>`;
    }

    return cells;
}

function renderWeekBadges() {
    const currentWeek = AppState.apply.currentWeek;
    const phases = [
        { label: 'Sem 1-4', start: 1, end: 4 },
        { label: 'Sem 5-8', start: 5, end: 8 },
        { label: 'Sem 9-12', start: 9, end: 12 }
    ];

    return phases.map(phase => {
        const isActive = currentWeek >= phase.start && currentWeek <= phase.end;
        const isComplete = currentWeek > phase.end;
        const isLocked = currentWeek < phase.start;

        let badgeClass = 'badge';
        if (isActive) badgeClass += ' badge-oxide';
        if (isComplete) badgeClass += ' badge-success';

        return `<span class="${badgeClass}" ${isLocked ? 'style="opacity: 0.5;"' : ''}>${phase.label}</span>`;
    }).join('');
}

function getPendingRecoveryDate() {
    const apply = AppState.apply;
    if (!apply.startDate) return null;

    const today = new Date();
    today.setHours(0, 0, 0, 0);

    const start = new Date(apply.startDate);
    start.setHours(0, 0, 0, 0);

    let cursor = new Date(start);
    let latestMiss = null;

    while (cursor < today) {
        const position = getProgramPositionForDate(cursor);
        if (!position || position.week > 12) break;

        const dateStr = cursor.toISOString().split('T')[0];
        const isRestDay = position.day === 7;

        if (!isRestDay && !apply.ledger[dateStr]) {
            latestMiss = dateStr;
        }

        cursor.setDate(cursor.getDate() + 1);
    }

    if (!latestMiss) return null;
    if (apply.recovery.lastAcknowledgedMissDate === latestMiss) return null;

    return latestMiss;
}

function getProgramPositionForDate(dateInput) {
    const apply = AppState.apply;
    if (!apply.startDate) return null;

    const start = new Date(apply.startDate);
    const date = new Date(dateInput);
    start.setHours(0, 0, 0, 0);
    date.setHours(0, 0, 0, 0);

    const diffDays = Math.floor((date - start) / (1000 * 60 * 60 * 24));
    if (diffDays < 0) return null;

    const adjustedDays = diffDays + apply.startDayOffset;
    return {
        week: Math.floor(adjustedDays / 7) + 1,
        day: (adjustedDays % 7) + 1
    };
}

function renderEntryType(type) {
    if (type === 'action') return 'Visualizacion + accion';

function escapeHtml(str) {
    if (!str) return '';
    return String(str)
        .replace(/&/g, '&amp;')
        .replace(/</g, '&lt;')
        .replace(/>/g, '&gt;')
        .replace(/"/g, '&quot;')
        .replace(/'/g, '&#039;');
}

function formatDate(dateStr) {
    const date = new Date(dateStr);
    return date.toLocaleDateString('es-ES', { weekday: 'short', day: 'numeric', month: 'short' });
}

function formatDateLong(dateStr) {
    const date = new Date(dateStr);
    return date.toLocaleDateString('es-ES', { day: 'numeric', month: 'long', year: 'numeric' });
}

function navigateTo(screenId) {
    // Hide all screens
    document.querySelectorAll('.screen').forEach(screen => {
        screen.classList.remove('active');
    });

    // Show target screen
    const target = document.getElementById('screen-' + screenId);
    if (target) {
        target.classList.add('active');
        AppState.currentScreen = screenId;
        window.scrollTo(0, 0);

        // Update UI based on screen
        if (screenId === 'apply-menu' && typeof initApplyMode === 'function') {
            initApplyMode();
        }
    }
}

const AppState = {
    currentScreen: 'home',
    theme: 'light',
    learnProgress: {
        unlearn: false,
        map: false,
        modules: {}
    },
    apply: createDefaultApplyState()
};

function mergeIdentityState(source) {
    const defaults = createDefaultIdentityState();
    const identity = source || {};

    return {
        ...defaults,
        ...identity,
        beliefs: Array.isArray(identity.beliefs)
            ? [identity.beliefs[0] || '', identity.beliefs[1] || '']
            : defaults.beliefs,
        dreamline: {
            ...defaults.dreamline,
            ...(identity.dreamline || {})
        },
        fearSetting: {
            ...defaults.fearSetting,
            ...(identity.fearSetting || {})
        }
    };
}

function mergeApplyState(source) {
    const defaults = createDefaultApplyState();
    const apply = source || {};

    return {
        ...defaults,
        ...apply,
        identity: mergeIdentityState(apply.identity),
        habits: {
            ...defaults.habits,
            ...(apply.habits || {})
        },
        recovery: {
            ...defaults.recovery,
            ...(apply.recovery || {})
        },
        ledger: apply.ledger || {}
    };
}

function loadState() {
    try {
        const saved = localStorage.getItem(STORAGE_KEY);
        if (saved) {
            const parsed = JSON.parse(saved);
            AppState.apply = mergeApplyState(parsed.apply);
            AppState.learnProgress = {
                ...AppState.learnProgress,
                ...(parsed.learnProgress || {})
            };
            if (parsed.theme) AppState.theme = parsed.theme;
            if (parsed.currentScreen) AppState.currentScreen = parsed.currentScreen;
        }
    } catch (e) {
        console.warn('[Winterlake] Failed to load state:', e);
    }

    recalculateDerivedState();

    if (typeof updateUI === 'function') updateUI();
}

function saveState() {
    try {
        localStorage.setItem(STORAGE_KEY, JSON.stringify({
            currentScreen: AppState.currentScreen,
            theme: AppState.theme,
            learnProgress: AppState.learnProgress,
            apply: AppState.apply
        }));
    } catch (e) {
        console.warn('[Winterlake] Failed to save state:', e);
    }
}

function recalculateDerivedState() {
    const apply = AppState.apply;

    if (!apply.startDate) {
        apply.currentWeek = 1;
        apply.currentDay = 1;
        apply.streak = 0;
        return;
    }

    const start = new Date(apply.startDate);
    const today = new Date();
    today.setHours(0, 0, 0, 0);
    start.setHours(0, 0, 0, 0);

    const daysSinceStart = Math.floor((today - start) / (1000 * 60 * 60 * 24));
    const adjustedDays = daysSinceStart + apply.startDayOffset;

    apply.currentWeek = Math.floor(adjustedDays / 7) + 1;
    apply.currentDay = (adjustedDays % 7) + 1;

    if (apply.currentWeek < 1) {
        apply.currentWeek = 1;
        apply.currentDay = 1;
    }

    if (apply.currentWeek > 12) apply.currentWeek = 12;

    if (apply.currentWeek >= 5) apply.habits.week5to8.unlocked = true;
    if (apply.currentWeek >= 9) apply.habits.week9to12.unlocked = true;

    apply.streak = calculateStreak();
}

function calculateStreak() {
    const ledger = AppState.apply.ledger;
    const entries = Object.keys(ledger).sort().reverse();

    if (entries.length === 0) return 0;

    const today = new Date();
    today.setHours(0, 0, 0, 0);
    const todayStr = today.toISOString().split('T')[0];

    let streak = 0;
    let checkDate = new Date(today);

    if (ledger[todayStr]) {
        streak = 1;
        checkDate.setDate(checkDate.getDate() - 1);
    } else {
        checkDate.setDate(checkDate.getDate() - 1);
        const yesterdayStr = checkDate.toISOString().split('T')[0];
        if (ledger[yesterdayStr]) {
            streak = 1;
            checkDate.setDate(checkDate.getDate() - 1);
        } else {
            return 0;
        }
    }

    while (true) {
        const dateStr = checkDate.toISOString().split('T')[0];
        if (ledger[dateStr]) {
            streak++;
            checkDate.setDate(checkDate.getDate() - 1);
        } else {
            break;
        }

        if (streak > 365) break;
    }

    return streak;
}

function resetApplyState() {
    if (!confirm('Resetear todo el progreso de Aplicar? Esta accion no se puede deshacer.')) {
        return false;
    }

    AppState.apply = createDefaultApplyState();
    saveState();
    return true;
}

# **Arqueidentidad — Introducción**

**Objetivo de este documento.** Presentar, de forma clara el mapa de **Arqueidentidad** en **seis fases** (I–VI). Para cada fase se explica **propósito**, **cómo funciona**, **técnicas o conceptos clave** y **por qué existe**.

---

## **Mapa general de las seis fases**

* **Fase I — Pre‑Tummoidentidad (Arquemétricas).**  
   Identificamos tu rol arquetípico con **arquemétricas†** y lo resumimos en una **“tarjeta de béisbol”**: qué se te da mejor, qué se te dificulta y qué errores son probables.

* **Fase II — Tummoidentidad I (Descubrimiento por Resonancias).**  
   **Tummo†** te prepara para **variar y seleccionar interpretaciones e identidades (conjuntos de interpretaciones y comportamientos)**. Exploramos los conceptos de **esferas y capas de resonancia†** y el **ciclo emocional†** para formular una **primera identidad heróica**.

* **Fase III — Interpretaciones Espontáneas y Controladas.** Probamos tu **héroe** con interpretaciones **espontáneas** (dionisíacas) y **controladas** (apolodionisíacas) en las cuatro esferas, y **ajustamos** el ciclo emocional con lo que mejor funcione.

* **Fase IV — Tummoidentidad II (Compresión & Microtemáticas).**  
   Refinamos el héroe y **comprimimos** técnicas (cambio de estado, meditación-concentración, journaling y visualización) para intervenir **microtemáticas†**: pequeños hábitos o creencias que mueven la **macro-vida**.

* **Fase V — Interpretaciones de Inmunización.**  
   **Subimos** de forma segura miedo/aversión para hacer **pruebas de estrés** en las cuatro esferas y consolidar **inmunización** frente a disonancias.

* **Fase VI — Reto Final.**  
   **Llevamos al límite** el control de interpretaciones para **máxima inmunización apolínea** y, luego, **liberamos** en **espontaneidad dionisíaca**.

*Los términos marcados con † aparecerán ampliados en glosario/notas al pie.*

---

## **Fase I — Pre‑Tummoidentidad (Arquemétricas)**

### **Propósito**

**Arqueidentidad I** es un proceso llamado **arquemétricas** que genera, como resultado, una **“tarjeta de béisbol”** de la persona: su **rol arquetípico** y cómo esto determina **fortalezas** y **debilidades** de su **temperamento**.

### **Cómo funciona**

1. **Arquemétricas**: se identifican **rasgos** y **tendencias** que ubican a la persona dentro de **roles** y **dinámicas** arquetípicas.

2. **Tarjeta tipo béisbol**: ficha sintética con **rol**, **sub‑rasgos**, **fortalezas**, **debilidades** y **errores comunes** asociados.

3. **Lectura temperamental**: se clarifica **cómo** el rol orienta decisiones, sesgos y acciones.

### **Técnicas y conceptos en esta fase**

* **Arquetipos comunes y errores (versiones luminosas y sombrías).**

  * **Rey / Esclavo**, **Egoísta / Místico** → **dinámicas generales**.

  * **Rigurosos / Rígidos / Ambiguos**, **Espontáneos / Inestables / Sobre‑complejidad** → **dinámicas específicas** ( ligadas a **cajas blancas** y **cajas negras)**.

* **Cuatro roles arquetípicos (con cara luminosa y sombra):**

  * **Shapers (Modeladores):** *Innovadores / Revolucionistas*.

  * **Harmonists (Armonizadores):** *Negociadores / Preservacionistas*.

  * **Strategists (Estrategas):** *Visionarios / Sobre-pensadores*.

  * **Workers (Hacedores/Constructores):** *Constructures / Sub-pensadores*.

**Nota sobre “cajas blancas / cajas negras”.** Se refiere a dinámicas de **apolíneo/dionisiaco** en procesos personales; su uso se detalla cuando corresponda a la selección de interpretaciones.

### **Por qué existe esta fase**

Para **nombrar** el **rol temperamental** y **anticipar** sus **errores típicos**; sirve de **base** para todo el trabajo posterior de variación y selección de interpretaciones e identidades.

---

## **Fase II — Tummoidentidad I (Descubrimiento por Resonancias)**

### **Propósito**

**Tummoidentidad I** es la **preparación teórica** para la **variación** y **selección de interpretaciones/identidades** de una **persona** en función de **resonancias**. Su fin es **incrementar** los **redes de interés y anti‑interés** y **crear un conjunto de identidades** a partir de interpretaciones que **resuenan más** con la persona.

### **Cómo funciona**

1. **Descubrimiento**: exploración de **esferas de resonancia** mediante **capas de resonancia**.

   * Esferas de resonancia: Eventos externos (percepción), eventos internos (interocepción) y eventos temperamentales (subconsciente). 

   * Capas de resonancia: Interpretaciónes físicas y psíquicas (quantia y qualia) y feedback subconsciente. 

2. **Modelado del ciclo emocional** para observar cómo cambian interés/anti‑interés según interpretaciones. 

3. **Primera conjetura de héroe/identidad**: se formula un **set de interpretaciones**, con **hypnos (hipnosis-reinterpretativa)**, **quest (identidades, habitos y creencias)**, **errores que el héroe no comete** y **comportamientos‑creencias** que sí tiene/hace.

4. **Principio de encarnación**: *“un héroe que podemos imaginar es un héroe que podemos instanciar en la vida cotidiana”.*

### **Técnicas y conceptos en esta fase**

* **Ciclos emocionales y resonancias/dissonancias; red de interés/anti‑interés.**

  * **Ciclo emocional:** `evento (quantia o qualia) → interpretación → deseo (feedback de inclinación) → acción → emoción (feedback de afinidad) → resultado/re‑start (resonancia o disonancia)`.

  * **Resultado/re‑start (feedback):**

    * *Buen feedback (a corto y largo plazo) \=* resonancia. 

    * *Mal feedback* \= disonancia (a corto y largo plazo), auto-regulación (a largo plazo) o re-interpretación (a corto plazo).

  * **Resultado/re‑start (interpretación):**

    * **aceptación‑reinterpretación o negación‑negativización** (exploración o imposición) → **creación‑construcción / auto‑destrucción** (diálogo interno de armonía o de conflicto).

* **Héroe (tummoidentidad, de héroe a héroe).**

  * **Primera conjetura** del **héroe/identidad** (set de interpretaciones), **hypnos**, **quest**, **errores que el héroe no comete** y **comportamientos‑creencias** que sí tiene/hace.

### **Por qué existe esta fase**

Para **mapear** cómo **interpretaciones** específicas **modulan** deseos, acciones, emociones y resultados; y para **fundar** una **identidad‑héroe** inicial **alineada** con resonancias reales.

---

## **Fase III — Interpretaciones Espontáneas y Controladas**

### **Propósito**

**Probar** las **resonancias** del héroe: primero con **interpretaciones espontáneas** que emergen **naturalmente**, y luego con **técnicas de control** de interpretaciones para **seleccionar** las que **mejor resuenan** con nosotros y con nuestros héroes/identidades.

### **Cómo funciona**

1. **Espontáneo (dionisíaco)**: se recorre, sin forzar, la aparición de interpretaciones en las esferas **corpomaterial**, **intelectual**, **emocional** y **sexocreativa por medio de intereses placenteros**.

2. **Controlado (apolíneo)**: se aplican **técnicas de control** para **modular/dirigir** interpretaciones y **elegir** las que **resuenan en las 4 esferas**.

3. **Comparación** de respuestas (espontáneas vs. controladas) y **ajustes** del set del héroe.

### **Técnicas y conceptos en esta fase**

* **Test de resonancias** del héroe en **cuatro esferas**.

* **Doble vía**: espontáneo (dionisíaco) y controlado (apolíneo), con foco en **selección**.

### **Por qué existe esta fase**

Para **contrastar** lo que **emerge** por sí mismo con lo que **podemos orientar**, y así **depurar** el **set de interpretaciones** operativas del héroe. En esta fase el héroe empieza a aclarar sus intereses significativos que determinarán su mitología y su ritualidad (re-interpretación significativa del mundo), las cuales son indispensables para un dialogo interno significativo (de armonía interna y no conflicto interno). 

---

## **Fase IV — Tummoidentidad II (Compresión & Microtemáticas)**

### **Propósito**

Construir un **mejor héroe** a partir de **nuevas interpretaciones** derivadas del **ciclo emocional** y **comprimir** técnicas para **intervenir microtemáticas**.

### **Cómo funciona**

1. **Microtemáticas**: son **temas** de **interés/anti‑interés** (nodos de la red/grafo) con **gran peso** sobre nosotros, que **generan disonancia** y **diálogo interno destructivo** por **reacciones emocionales/impulsivas** y **interpretaciones** que **hacen más daño que bien**.

   * Estas **grandes temáticas** (que se repiten en problemas o eventos significativos) **aparecen** como **micro‑comportamientos** y **micro‑creencias** (algunas **conscientes**, otras **subconscientes**) que dejamos **gobernar** nuestra vida **sin cuestionamiento**.

   * Una vez nuestros intereses son resonantes y nuestros anti-intereses disonantes, nos alejamos de todo aquello que supone una imposición y un dialogo interno conflictivo, para acercarnos a aquellos intereses que genuinamente nos entusiasman, así como el amante se entusiasma al contemplar al amado, y se transforma radicalmente para ser digno del amado (siendo el amado la representación mística del universo como movimiento infinito y eterno). 

2. **Compresión del héroe**: **cambio de estado corporal**, **meditación‑concentración**, **journaling** y **visualización**, **comprimidos** en **pocos segundos** para atender **micro‑comportamientos/creencias** que afectan la **macro‑vida**.

### **Técnicas y conceptos en esta fase**

* **Derivación** de nuevas interpretaciones desde el **ciclo emocional**.

* **Compresión** de **(1) cambio de estado corporal**, **(2) meditación‑concentración**, **(3) journaling**, **(4) visualización**.

* **Intervención** en **microtemáticas** (nodos de interés/anti‑interés con alto peso).

## **Glosario**

* Hábito — Un comportamiento que resuelve un problema o aporta significado, y que se repite en el tiempo de manera consistente. 

* Creencia — Una predicción (narrativa/historia) que resuelve un problema o aporta significado, que se soporta en evidencia que refuerza la predicción, y que se repite en el tiempo de manera consistente.

* Identidad — Conjunto de hábitos y creencias para resolver problemas, y crear significados; puede ser impuesta y detonar diálogos internos conflictivos, o puede ser descubierta y generar diálogos internos armoniosos. 

* **Arquemétricas** — Proceso para identificar el **rol arquetípico** y resumirlo en una **tarjeta tipo béisbol** (rol, fortalezas, debilidades, errores comunes). Base de **Pre‑Tummoidentidad**.

* **Tummo / Tummoidentidad** — Preparación teórica para la **variación y selección de interpretaciones** basadas en **resonancias**, con el fin de **incrementar** las **redes de interés/anti‑interés** y **crear conjuntos de identidades**.

* Resonancia e intereses — Alineación entre interpretaciones y feedback subconsciente. Define nuestros intereses, nos energiza y nos atrae. 

* Disonancia y anti-intereses — Desalineación entre interpretaciones y feedback subconsciente. Define nuestros anti-intereses, que nos drenan y nos repulsan; sin embargo se mezclan con nuestro intereses genuinos en forma de un "interés" impuesto, o un interés genuino rechazado por imposiciones sociales.

* **Esferas de resonancia** — Tres focos: **eventos externos** (percepción), **eventos internos** (interocepción) y **eventos temperamentales** (subconsciente).

* **Capas de resonancia** — **Interpretaciones físicas** y **psíquicas** (*quantia* y *qualia*) y **feedback subconsciente**.

* **Esferas (4)** — **Corpomaterial**, **Intelectual**, **Emocional** y **Sexocreativa** (usadas para testear interpretaciones).

* **Ciclo emocional** — `evento (quantia/qualia) → interpretación → deseo (feedback de inclinación) → acción → emoción (feedback de afinidad) → resultado/re‑start (resonancia o disonancia)`; permite decidir **aceptación‑reinterpretación** vs **negación‑negativización**.

* **Resonancia / Disonancia** — Resultado de la interacción entre evento e interpretación que **aumenta** o **disminuye** afinidad, interés y coherencia del diálogo interno.

* **Red/Grafo de interés y anti‑interés** — Mapa de **nodos** y **pesos** que representan lo que atrae/repulsa; guía la selección de interpretaciones e **indicadores de progreso**.

* **Héroe / Identidad** — **Set de interpretaciones** instanciables en la vida cotidiana (incluye **hypnos**, **quest**, **errores que no comete** y **comportamientos‑creencias** que sí tiene/hace).

* **Hypnos (hipnosis‑reinterpretativa)** — Técnica para **anclar** reinterpretaciones y encarnarlas en el **héroe/identidad**.

* **Quest (identidades, hábitos y creencias)** — Definición operativa de **metas**, **hábitos** y **creencias** del héroe; guía acciones consistentes. Creación de misiones apolíneas (para resolver problemas) y misiones dionisiacas (para crear mitologías y rituales significativos).

* **Cajas blancas / Cajas negras** — Dinámicas de **apolíneo/dionisíaco** aplicadas a las formas de resolver problemas, a través de la fluidez o la asertividad.

* **Apolíneo / Dionisíaco** — **Apolíneo**: orden, medida, control consciente; **Dionisíaco**: espontaneidad, juego, flujo.

* **Interpretaciones espontáneas** — Predicciones que **emergen naturalmente** en las cuatro esferas tras usar técnicas que inducen jubilo (énfasis dionisíaco).

* **Interpretaciones controladas** — Predicciones **dirigidas** mediante técnicas de control en las cuatro esferas para **seleccionar** las que mejor resuenan (énfasis apolíneo).

* **Microtemáticas** — **Nodos** (temas, hábitos o creencias) con **alto peso** que, de ser impuestos, generan **disonancia** y **diálogo interno destructivo** si no se intervienen. Y de ser espontáneos, si no se cultivan, alejan de la resonancia y el diálogo interno significativo. 

* **Compresión (del héroe)** — Paquete **ultra‑breve** de técnicas: **cambio de estado corporal**, **meditación‑concentración**, **journaling** y **visualización** ejecutado en **segundos** para intervenir **microtemáticas** en tiempo real.

* **Inmunización (apolodionisíaca)** — **Pruebas de estrés** (miedo/aversión elevados de forma segura) para entrenar **voluntad** y **flexibilidad** y así **evitar disonancias** a voluntad.

* **Villano externo / interno / filosófico** — Figuras que estructuran la **mitología personal**; ayudan a delimitar **conflictos** y **criterios** de elección.

* **Mitología / Ritualidad** — Re‑interpretación significativa del mundo (mitología) y **prácticas** que la encarnan (ritualidad) orientadas por la **red de interés**.

* **Control lúcido** — Técnicas para **mantener agencia** sobre interpretaciones y estados oníricos. 

* **Disonancias genéticas y meméticas** — Orígenes **biológicos** y **culturales** de ciertas disonancias que se buscan **inmunizar**.
